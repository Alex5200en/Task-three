# Task-three

Источники событий — бизнес-сервисы (заказы, корзина, акции) генерируют доменные события. Они не знают ничего про push — только публикуют факт.

Kafka / RabbitMQ — брокер разделяет источники и потребителей. При расширении список триггеров для пушей не требует изменений в бизнес-сервисах.

Notification Service — подписывается на топики, обогащает событие шаблоном сообщения, проверяет настройки пользователя (разрешил ли пуши, часовой пояс), выполняет дедупликацию.

User Profile DB — хранит device token пользователя (FCM token или APNs token), полученный при регистрации приложения.

Scheduler Service — запускает отложенные и периодические задачи (например, напоминание о брошенной корзине через 24 часа).

Push Gateway Service — единая точка отправки, управляет retry-логикой, батчингом и статусами доставки.

FCM / APNs — сторонние провайдеры Google и Apple, через которые пуш физически доставляется на устройство.
